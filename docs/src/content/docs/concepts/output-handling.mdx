---
title: Output handling
description: Receive PDF bytes in memory, as a stream, written to disk, or stored server-side.
---

The Rust SDK gives you four ways to handle a rendered PDF: in-memory bytes, an async stream, a write-to-disk helper, or a stored document on the server. Pick the one that matches how your code consumes the output.

## In-memory (`render.pdf`)

```rust
let pdf = client.render.pdf(ProjectModeInput {
    project: "billing".into(),
    template: "invoice".into(),
    version: Some("1.0.0".into()),
    data: json!({ "invoiceNumber": "INV-001" }),
    ..Default::default()
}).await?;
// pdf: bytes::Bytes — write, upload, return as-is.
```

Best for documents that fit comfortably in process memory.

## Streaming (`render.pdf_stream`)

For large PDFs or piping straight to an HTTP response or object store, use `pdf_stream` and let the SDK keep memory bounded. The returned `PdfByteStream` implements `futures_core::Stream<Item = Result<Bytes, Error>>`:

```rust
let mut stream = std::pin::pin!(client.render.pdf_stream(input).await?);
// Consume with `StreamExt::next` or `poll_fn`.
```

## Write to file (`render_to_file`)

The free-standing helper streams bytes directly to disk with bounded memory. Parent directories are created automatically:

```rust
poli_page::render_to_file(&client, input, "./out.pdf").await?;
```

## Server-side storage (`render.document`)

When you want the PDF to live on Poli Page's servers (for later download, preview, or thumbnails), use `render.document`. It returns a `DocumentDescriptor` with `document_id`, `presigned_pdf_url`, and a `download_pdf()` method.

```rust
let doc = client.render.document(input).await?;
let pdf = doc.download_pdf().await?;
```

`presigned_pdf_url` has a 15-minute TTL. If it expires, call `client.documents.get(&document_id)` to refresh.

## Example

```rust
use poli_page::{PoliPage, ProjectModeInput};
use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = PoliPage::new(std::env::var("POLI_PAGE_API_KEY")?)?;

    poli_page::render_to_file(
        &client,
        ProjectModeInput {
            project: "billing".into(),
            template: "invoice".into(),
            version: Some("1.0.0".into()),
            data: json!({ "invoiceNumber": "INV-001" }),
            ..Default::default()
        },
        "./invoices/INV-001.pdf",
    ).await?;

    Ok(())
}
```
