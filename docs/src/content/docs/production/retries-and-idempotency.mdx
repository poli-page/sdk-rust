---
title: Retries and idempotency
description: How the Rust SDK retries transient failures and how to make your render calls safe to retry.
---

The Poli Page Rust SDK retries transient failures automatically. Render operations carry an auto-generated UUID v4 idempotency key, so retries do not produce duplicate stored documents on the server.

## Default retry policy

- `Error::Connection`, `Error::Timeout`, 5xx responses, and 429 (`RateLimited`) are retried.
- Up to `max_retries` attempts (default `2` — three total attempts).
- Exponential backoff starting at `retry_delay` (default 500 ms), doubling each attempt, plus jitter.
- Honors the `Retry-After` response header when present.
- `Error::Aborted` is never retried.

## Disabling retries

Set `max_retries(0)` on the builder to disable retries entirely:

```rust
use poli_page::PoliPage;

let client = PoliPage::builder()
    .api_key(std::env::var("POLI_PAGE_API_KEY").unwrap())
    .max_retries(0)
    .build()
    .unwrap();
```

## Idempotency

Every render request includes a UUID v4 `Idempotency-Key` header generated by the SDK. The API treats requests with the same key as the same operation, so retries from inside the SDK never create duplicate stored documents.

If you orchestrate your own retries on top of the SDK (not recommended), supply a deterministic key on the input:

```rust
use poli_page::ProjectModeInput;
use serde_json::json;

let input = ProjectModeInput {
    project: "billing".into(),
    template: "invoice".into(),
    version: Some("1.0.0".into()),
    data: json!({ "invoiceNumber": "INV-001" }),
    idempotency_key: Some("invoice-INV-001-v1".into()),
    ..Default::default()
};
```

Same input + same key → same stored document. Different key with same data → new document.

## Example

```rust
use poli_page::{PoliPage, ProjectModeInput};
use serde_json::json;
use std::time::Duration;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = PoliPage::builder()
        .api_key(std::env::var("POLI_PAGE_API_KEY")?)
        .max_retries(4)
        .retry_delay(Duration::from_millis(250))
        .build()?;

    let _pdf = client.render.pdf(ProjectModeInput {
        project: "billing".into(),
        template: "invoice".into(),
        version: Some("1.0.0".into()),
        data: json!({ "invoiceNumber": "INV-001" }),
        ..Default::default()
    }).await?;

    Ok(())
}
```
