Every fallible operation in the crate returns `Result<T, poli_page::Error>`. The error enum splits cleanly into SDK-internal variants (no wire `code`) and API-status variants (carrying `status`, `code`, `message`, and `request_id`).

## The Error enum

```rust
use poli_page::{Error, PoliPage, ProjectModeInput};

#[tokio::main]
async fn main() {
    let client = PoliPage::new(std::env::var("POLI_PAGE_API_KEY").unwrap()).unwrap();

    match client.render.pdf(ProjectModeInput::default()).await {
        Ok(pdf) => { let _ = pdf; }
        Err(err) => match err {
            Error::Auth { request_id, .. } => eprintln!("auth failed; req {:?}", request_id),
            Error::RateLimited { .. } => eprintln!("rate limited"),
            other => eprintln!("{}: {}", other.code(), other),
        },
    }
}
```

`Error` is `#[non_exhaustive]` — always include a catch-all arm so future variants don't break your match.

## Convenience predicates

For cross-language parity the enum exposes `is_auth_error()`, `is_rate_limit_error()`, `is_validation_error()`, `is_network_error()`, and `is_retryable()`. Use them when you don't need to bind the inner fields.

## Example

```rust
use poli_page::{Error, PoliPage, ProjectModeInput};
use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = PoliPage::new(std::env::var("POLI_PAGE_API_KEY")?)?;

    let input = ProjectModeInput {
        project: "billing".into(),
        template: "invoice".into(),
        version: Some("1.0.0".into()),
        data: json!({ "invoiceNumber": "INV-001" }),
        ..Default::default()
    };

    match client.render.pdf(input).await {
        Ok(_pdf) => {}
        Err(Error::RateLimited { request_id, .. }) => {
            eprintln!("rate limited (req {:?}) — back off", request_id);
            return Ok(());
        }
        Err(err) => return Err(err.into()),
    }
    Ok(())
}
```
